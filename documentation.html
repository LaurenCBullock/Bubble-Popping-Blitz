Game Documentation

Process

I decided to base this game off an old Disney fairy’s game. I started off with the base code from circle blast to have the basic game layout. Then I created a basic 2D array to make the grid system. I created the bubbles and droplets/bullets next. The collisions between the two were problematic for a while, the first time it was because of a code error and the second time it was because I was implementing size for the bubbles. In implementing size for bubbles the collision had to be updated so if the size was smaller than 4 it would just increase the size and if it was smaller it would just grow. The rest of the game was easier to complete.

Project Requirements

The game has been tested and is fully functional, it throws no error and it uses PixiJS. The game also uses local storage to store the highscore. The game is easy to use and has an old-school look. The page webpage the game is on has valid HTML and CSS. The Howler library is used to add sound effects to the game to make it more immersive and enjoyable. The code is clean and has nicely commented chunks.

WebSite Documentation

Process

In creating this page, I started with writing all the content I wanted to convey into a word document. Then I decided how I wanted to lay it out on a page. Since the information was simple I figured a single column layout would suffice. After that I chose a color scheme that would fit the website. Next I started coding it. Finally, I checked it over to make sure everything was connected and validated and uploaded it to Banjo.rit.edu.
